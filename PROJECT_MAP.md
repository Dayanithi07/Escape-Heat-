## 🔍 Detailed Component Audit

### 1. Next.js Frontend (`escape-heat/`)
* **What is Complete:**
  - Full routing structure using Next.js App Router (Landing, Dashboard, Heatmap, Assistant, Recommendations, Settings, Profile).
  - Modern, responsive styling with Tailwind CSS v4, dark/light theme options, and custom animations.
  - Interactive Leaflet maps showing mock heat zones and point of interest (POI) markers.
  - High-performance dashboards using Chart.js visualizations.
* **What is Pending for Production:**
  - **Dynamic Fetching**: Replace static array returns in `escape-heat/lib/mock-data/` with API hooks (e.g., SWR, TanStack Query) querying `http://localhost:8000/api/v1/`.
  - **Auth Integration**: Connect to Supabase Auth. Set up a global auth state provider and route-level authorization guards for protected pages (e.g., `/profile`, `/settings`, `/assistant`).
  - **Error Boundaries**: Add React Error Boundaries and skeleton loader states (`components/ui/LoadingSkeleton.tsx`) to handle slow external APIs.

### 2. FastAPI Backend (`backend/`)
* **What is Complete:**
  - Web server boilerplate (`run.py`, `app/main.py`), CORS headers, exception handlers, and configuration loaders.
  - Complete REST router layout mapping public/private endpoints.
  - Pydantic schema validation for requests and responses.
* **What is Pending for Production:**
  - **Database Repository Implementations**: Refactor repositories (like `UserRepository` in `app/repositories/user_repository.py`) to connect to the live PostgreSQL/Supabase instance instead of using local in-memory arrays.
  - **Live Weather Integration**: Replace `app/services/weather_service.py` mock responses with actual HTTP integrations to the Open-Meteo & NASA POWER APIs.
  - **Live Gemini Assistant**: Implement the Gemini API client inside `app/services/ai_chat_service.py` using the Google AI SDK or the Agent Development Kit (ADK) to process real user prompts based on live weather data.
  - **Access Tokens**: Implement JWT verification middleware inside dependencies (`app/api/deps.py`) to check tokens generated by Supabase Auth.

### 3. Database Layer (`database/`)
* **What is Complete:**
  - Twelve sequential migrations creating all relational tables, types, enums, triggers, and indices.
  - Advanced PostgreSQL functions (RPCs) for atomic coordinate rounding, dashboard status aggregations, GDPR deletions, and cache cleanup.
  - Row Level Security (RLS) policies defining secure, multi-tenant boundaries.
  - Automated verification script checking over 25 schema conditions.
* **What is Pending for Production:**
  - **Production Deployment**: Provision a live Supabase project and execute the migrations in sequence.
  - **Cron Automation**: Set up a PostgreSQL cron-trigger (or external serverless function) to periodically call the `cleanup_expired_cache()` RPC to prevent database bloat.

## 🚀 Step-by-Step Production Roadmap

### Phase 1: Database Activation & Repository Wiring
1. Deploy the PostgreSQL database migrations using the Supabase SQL editor or CLI.
2. Install a PostgreSQL driver (`psycopg2-binary` or `asyncpg`) in the FastAPI backend.
3. Replace memory-backed files in `backend/app/repositories/` with SQL-based implementations. For example:
   ```python
   # app/repositories/user_repository.py
   # Replace mock list with database queries:
   # select * from public.profiles where email = :email
   ```

### Phase 2: Live Authentication Setup
1. Configure Supabase Auth in the frontend using the `@supabase/supabase-js` client SDK.
2. Protect frontend pages by wrapping routes with the Supabase session listener.
3. Update `backend/app/api/deps.py` to extract the JWT Bearer token from the `Authorization` header, verify it using Supabase's public keys, and load the user ID.

### Phase 3: External API & AI Integrations
1. **Weather**: Implement `httpx.AsyncClient` calls to the Open-Meteo API in `backend/app/services/weather_service.py`. Include the local environmental cache check:
   - Call the `get_cached_environment(lat, lon)` database RPC first.
   - If not cached (or expired), call Open-Meteo, then save the response via the `upsert_environmental_cache` RPC.
2. **AI Assistant**: Setup Google AI Studio / Gemini API.
   - Feed user prompts along with the current weather JSON and profile hazards into a system prompt.
   - Return structured text with recommendations and suggested follow-ups.

### Phase 4: Frontend-to-Backend Wiring
1. Replace imports of `lib/mock-data` in the Next.js pages with dynamic API calls using a wrapper client:
   ```typescript
   // Example React Hook
   const { data, error } = useSWR('/api/v1/weather/current?lat=...&lon=...', fetcher);
   ```
2. Configure frontend environment variables (`NEXT_PUBLIC_API_URL`) to point to the FastAPI backend.

### Phase 5: Testing, Hardening & Scaling
1. **Caching**: Setup a Redis instance to cache repeated geospatial coordinate queries.
2. **Rate Limiting**: Configure FastAPI rate limiters (e.g., `slowapi`) to protect backend endpoints.
3. **Tests**: Update backend test files in `backend/tests/` to mock the external API calls using `pytest-mock` or `responses`.
4. **Environment Hardening**: Switch `DEBUG=False` in `.env`, configure production CORS constraints, and enforce SSL/HTTPS redirects.

## 🛠️ Technical Integration Recipes

To speed up development, use the following blueprints to integrate the core live features:

### Recipe A: Live Weather API (Open-Meteo & Cache check)
Update `backend/app/services/weather_service.py` to fetch from live endpoints with coordinate normalization:

```python
import httpx
from typing import Dict, Any

class WeatherService:
    async def get_current_weather(self, lat: float, lon: float) -> Dict[str, Any]:
        # Round coordinates to 3 decimals to improve cache hit rates (~100m accuracy)
        r_lat, r_lon = round(lat, 3), round(lon, 3)
        
        # 1. (Optional) Query public.environmental_cache via DB Client
        # 2. If Cache Miss, fetch from Open-Meteo:
        url = "https://api.open-meteo.com/v1/forecast"
        params = {
            "latitude": r_lat,
            "longitude": r_lon,
            "current": ["temperature_2m", "relative_humidity_2m", "apparent_temperature", "uv_index", "wind_speed_10m"],
            "timezone": "auto"
        }
        
        async with httpx.AsyncClient() as client:
            response = await client.get(url, params=params)
            response.raise_for_status()
            data = response.json()
            
        # Parse into standard response schema
        current = data.get("current", {})
        return {
            "location": f"Coords: {r_lat}, {r_lon}",
            "temperature": current.get("temperature_2m"),
            "apparent_temperature": current.get("apparent_temperature"),
            "humidity": current.get("relative_humidity_2m"),
            "uv_index": current.get("uv_index"),
            "wind_speed": current.get("wind_speed_10m"),
        }
```

### Recipe B: Connect Gemini Pro AI with Context
Update `backend/app/services/ai_chat_service.py` to use the official Google Generative AI SDK:

```python
import google.generativeai as genai
from typing import Dict, Any, Optional

class AIChatService:
    def __init__(self, api_key: str):
        genai.configure(api_key=api_key)
        self.model = genai.GenerativeModel('gemini-1.5-flash')

    async def chat(
        self,
        message: str,
        context: Optional[Dict[str, Any]] = None,
        user: Optional[dict] = None
    ) -> Dict[str, Any]:
        
        # Build a robust system context
        weather_info = context.get("weather", {}) if context else {}
        profile_info = user.get("preferences", {}) if user else {}
        
        system_prompt = (
            f"You are Escape AI, an expert urban heat safety assistant. "
            f"Current weather context: Temp={weather_info.get('temperature')}°C, "
            f"Humidity={weather_info.get('humidity')}%, UV Index={weather_info.get('uv_index')}. "
            f"User Profile details: health_conditions={profile_info.get('conditions', 'None')}. "
            f"Provide highly actionable, empathetic heat safety advice based strictly on "
            f"scientific guidance. Limit responses to 3 bullet points. No weather predictions."
        )
        
        # Query Gemini
        response = self.model.generate_content(
            contents=[system_prompt, message]
        )
        
        return {
            "response": response.text,
            "sources": ["Open-Meteo API", "Heat Risk Models"],
            "suggested_questions": ["When is it safe to go outside?", "Show cooling centers nearby"]
        }
```

### Recipe C: Verify JWT in FastAPI via Supabase Public Keys
Verify JWTs in `backend/app/api/deps.py`:

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
import jwt # PyJWT

security = HTTPBearer()

async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security)
) -> dict:
    token = credentials.credentials
    try:
        # Decode and verify the JWT token using your Supabase JWT Secret 
        # (For production, retrieve the public keys from Supabase JWKS endpoint)
        payload = jwt.decode(token, "YOUR_SUPABASE_JWT_SECRET", algorithms=["HS256"], audience="authenticated")
        user_id: str = payload.get("sub")
        if not user_id:
            raise HTTPException(status_code=401, detail="Invalid token payload")
        return {"id": user_id, "email": payload.get("email")}
    except Exception as e:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail=f"Could not validate credentials: {str(e)}"
        )
```
